EC2 Instance Creation Guide

  1. Sign in to AWS Console
    • Go to AWS Management Console.
    • Sign in with your credentials.
  2. Navigate to EC2 Dashboard
    • In the AWS Management Console, type "EC2" in the search bar and select it.
  3. Launch Instance
    • Click the "Launch Instance" button.
  4. Choose an Amazon Machine Image (AMI)
    • Select an AMI that meets your requirements. Common choices include Amazon Linux 2, Ubuntu, or Windows Server.
  5. Choose an Instance Type
    • Select the instance type that fits your use case. For example, t2.micro is commonly used for free tier eligibility.
  6. Configure Instance
    • Number of Instances: Usually 1, but you can specify more.
    • Network: Choose a VPC (Virtual Private Cloud) or use the default one.
    • Subnet: Select a subnet (the default one is usually fine).
    • Auto-assign Public IP: Choose "Enable" if you need a public IP address.
    • Advanced Details: You can add user data scripts here for initialization tasks.
  7. Add Storage
    • Modify the storage as needed. The default is often an 8 GB EBS (Elastic Block Store) volume.
    • You can add more volumes if required.
  8. Add Tags
    • Add tags to manage your instance. For example, add a tag with a key of "Name" and a value of your instance name.
  9. Configure Security Group
    • Create a new security group or select an existing one.
    • Define rules for inbound and outbound traffic, such as allowing SSH (port 22) for Linux or RDP (port 3389) for Windows.
  10. Review and Launch
    • Review all the configurations.
    • Click on "Launch".
  11. Choose or Create a Key Pair
    • Select an existing key pair or create a new one. This key pair will allow you to SSH into your instance.
    • Download the private key (.pem file) and store it securely.
  12. Launch the Instance
    • After selecting your key pair, click "Launch Instances".
    • You will see a confirmation page with your instance ID.
  13. Access Your Instance
    • Use the .pem file to SSH into your EC2 instance if it's Linux-based, or use RDP if it's Windows-based.
    • Example SSH command: 
      ssh -i "your-key.pem" ec2-user@your-public-ip
  14. Monitor and Manage
    • Go back to the EC2 Dashboard to monitor your instance, view metrics, and manage it.
